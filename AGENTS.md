## 🏗️ 1. BaseAgent: 生命の設計図
すべてのエージェントは `BaseAgent` を継承し、単一の責務を持つ。

### ライフサイクル・メソッド
1. **`initialize(): Promise<void>`**
   - 設定(`Core.config`)と秘匿情報(`Core.getEnv`)の読み込み。
   - Zod による依存コンポーネントの存在確認。不備があれば `process.exit(1)`。
2. **`plan(): Promise<ActionPlan>`**
   - 直近の `Signal` とマーケットコンテキスト（JQuants等）を解析。
   - 実行すべきタスクの優先順位付けとリソース配分の決定。
3. **`execute(): Promise<ActivityResult>`**
   - `use_cases` または `providers` を介した実際の演算・外部通信。
   - 冪等性を保ち、途中でクラッシュしても再開可能な設計を徹底。
4. **`reflect(): Promise<void>`**
   - 実行ログをメタ解析し、LLM経由で翌日のプロンプト重み付けや閾値を自律調整。

## 🎭 2. 16 Parallel Agents (戦略マトリックス)
エージェントは16体の特化型人格として定義され、時間軸と戦略を分散させることでポートフォリオの安定化を図る。

| ID | 名称 | 時間軸 | 主な戦略 & 使用データソース |
| :--- | :--- | :--- | :--- |
| **A-01** | **PEAD-S** | 短期 (1-3日) | 決算サプライズ(SUE)に対する初動捕捉。JQuants Statements。 |
| **A-02** | **PEAD-M** | 中期 (1-4週) | 決算情報の非対称性解消。ポスト決算ドリフトの追跡。 |
| **A-03** | **Event-Buyback** | 中期 | 自社株買い発表後の需給改善狙い。EDINET / TDnet。 |
| **A-04** | **Event-MSCI** | 短期 | 指数構成銘柄の入替に伴うリバランス需要の先回り。 |
| **A-05** | **Sent-X** | リアルタイム | SNS (X/Grok) 上の熱狂とセンチメントの数値化。 |
| **A-06** | **Sent-News** | 日次 | 適時開示・ニュースの行間読解による定性スコアリング。 |
| **A-07** | **Arb-Cross** | 超短期 | 銘柄間の相関崩れを利用したペアトレード。Tickデータ。 |
| **A-08** | **Macro-Estat** | 長期 | 景気動向指数、e-STAT 統計によるセクターローテーション。 |
| **A-09** | **Tech-ATR** | 継続的 | ボラティリティ監視による動的な損切り・利確位置の調整。 |
| **A-10** | **Liquidity-Guard** | リアルタイム | 板情報監視による大口注文・見せ板の検知。 |
| **A-11** | **Kelly-Optimizer** | 日次 | 全戦略の勝率・期待値から、ケリー基準で最適配分を計算。 |
| **A-12** | **Regime-Detector** | 日次 | 市場が「凪」か「嵐」か。レバレッジ倍率の統括決定。 |
| **A-13** | **TOB-Scanner** | 不定期 | 公開買付・合併ニュースの即時解析と裁定判断。 |
| **A-14** | **Value-Deep** | 長期 | 資産内容と PBR-ROE マトリックスによる割安放置銘柄の発掘。 |
| **A-15** | **Flow-Tracker** | 週次 | 投資部門別売買動向による大口主体（外国人・信託他）の追随。 |
| **A-16** | **Meta-Reviewer** | 夜間 | 全エージェントの失敗/成功を総括し、システム全体の行動を補正。 |

## 🔄 3. シグナル・プロトコル (エージェント間通信)
エージェント同士は直接結合せず、`Signal` と呼ばれる不変なイベントを介して情報を共有する。

### Signal スキーマ (Zod)
```typescript
const SignalSchema = z.object({
  id: z.string().uuid(),
  sourceAgent: z.string(),
  type: z.enum(['ALPHA', 'RISK', 'NEWS', 'ORDER']),
  payload: z.record(z.any()),
  timestamp: z.date(),
  reliability: z.number().min(0).max(1), // 信頼度スコア
});
```

### 通信フロー
1. **アルファ検知**: `A-01` が SUE 2.0 超えを検知。
2. **シグナル発行**: `ALPHA` シグナルをブロードキャスト。
3. **リスク検証**: `A-09` が現在のボラティリティを確認、`RISK` シグナルで許可/否認を返答。
4. **資金配分**: `A-11` が適切なロット数を決定。
5. **最終執行**: `OrderUseCase` が auカブコム API 経由で発注。

## 🛡️ 4. 厳格な動作規定
- **Dependency Inversion (DIP)**: エージェントは `JQuantsProvider` を直接インスタンス化してはならない。必ずコンストラクタでインターフェースを受け取り、モックテストを可能にすること。
- **Immutability**: エージェントが受信した `Signal` や `Config` は絶対に書き換えてはならない。
- **Fail-Fast**: 通信エラーやバリデーション失敗時、中途半端なリトライは行わず、即座に終了して通知を行うこと。
