# Alpha Discovery Runbook (sequence準拠)

本ワークフローは `/docs/diagrams/sequence.md` の参加者と遷移を実装用手順に落としたものです。  
目的は、入力テンプレート固定による探索の偏りを解消し、`antigravity` の agentic 実行で初期入力を毎回再構成して直交アルファ探索を継続することです。

## 参加者

- 人間: 要件入力
- 統括: オーケストレータ
- 記憶: 長老
- データ: 市場データ基盤エンジニア
- 分析: クオンツ研究者
- 執行: 執行エージェント

## 実行フロー

### フェーズ1: 入力と探索
1. 人間が要件を入力する。
2. 統括が長老から履歴を取得する。
3. 長老はシードと禁止領域を返す。
4. 統括は `antigravity` agentic を使い、次を束ねて初期入力を動的生成する。  
   - 現要件  
   - 履歴シード  
   - 禁止領域  
   - 納入条件  
5. 統括はアイデア生成を実行し、候補を長老へ保存する。
6. 統括はデータ作成を依頼し、PIT整合と欠損補完済みデータを受領する。
7. 統括はデータ納入条件を判定する。未達ならデータ作成へ戻す。
8. 達成時はデータ版と前処理条件を長老へ保存する。

### フェーズ2: 評価と判定
9. 統括はクオンツ研究者へ候補式とデータセットを渡す。
10. クオンツ研究者は基盤モデル選定、適応方針設計、因子探索、候補精査、共最適化、バックテストを実行する。  
    このとき、相場状態に応じて次を同時に適応設計する。  
    - 売買ロジック（エントリー/イグジット/サイズ/回転率/コスト反映）  
    - アルファ（実行AST式と経済的意味）  
    - トレード戦略（ユニバース/配分/リスク制約）
11. クオンツ研究者は採否、主要指標、モデル構成に加えて、上記3点の最終仕様を返す。
12. 統括は納入条件を判定する。  
    - データ起因未達: データ作成へ戻す。  
    - モデル起因未達: モデル選定へ戻す。  
13. 達成時は検証結果とモデル構成を長老へ保存する。

### フェーズ3: 執行
14. 採用なら発注可否を最終確認し、可なら執行エージェントへ注文生成を依頼する。
15. 執行結果または不採用理由を長老へ保存する。

## antigravity agentic 入力仕様

固定テンプレート入力は禁止。統括は毎回、以下の入力を生成して分析へ渡す。

- `mission`: 今回の探索目的
- `constraints`: 禁止領域、容量、期限
- `memory_context`: 長老由来の履歴シード、既存採否理由
- `data_contract`: 必須カラム、PIT条件、欠損許容、納入閾値
- `evaluation_contract`: Sharpe/IC/MDD/相関などの採否基準
- `return_path`: 未達時の戻り先（データ起因/モデル起因）

分析への返却物は、説明文だけでなく以下の4点セットを必須とする。  
- 実行AST式  
- 売買ロジック仕様  
- ポートフォリオ/配分仕様  
- Net-of-Costs 検証結果

## 実行コマンド

- 探索: `cd ts-agent && bun run experiments:alpha-discovery`
- 検証: `cd ts-agent && bun run pipeline:ab`
- 型検査: `cd ts-agent && bun run typecheck`
