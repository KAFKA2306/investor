# ArXiv 日本語レポート作成スキル

このスキルは、自律型投資エージェントが生成する研究成果を、「エグゼクティブ・サマリー」および「4-Tier Evaluation Framework」に基づいた高品質な日本語レポートに変換するための標準プロトコルを定義します。

## 1. 基本原則 (Core Principles)

- **言語**: 日本語を第一言語とする。専門用語については、文脈に応じて英語を併記（例：情報係数 (Information Coefficient)）することを推奨する。
- **トーン**: 投資委員会（Executive Committee）や上級幹部に提出することを想定した、専門的かつ端的な文体とする。
- **透明性**: 戦略仮説、実証データ、統計的有意性、および「自己レビュー（Self-Review）」を必須項目とする。

## 2. 標準構成 (Standard Layout)

### 2.1 エグゼクティブ・サマリー (Executive Summary)
戦略の要点、アルファの源泉、および最終的な検証結果（承認/却下）を端的に記述する。

### 2.2 検証結果 (KPIs)
以下の4つのKPIを必ず含め、表形式で提示する。
- **年間超過収益 (Alpha / 年率)**
- **リスク調整後収益 (Sharpe Ratio)**
- **予測方向正確基準 (Directional Accuracy)**
- **統合推論スコア (Reasoning Score: RS)**

### 2.3 統計的有意性 (Tier 1 Validation)
- **t-Stat**: 統計的な確信度。
- **p-Value**: 偶然である確率（有意水準）。
- **Information Coefficient (IC)**: 予測指標と実際の結果の相関。

### 2.4 自己レビュー・自己評価 (Self-Review Protocol)
エージェント自身による客観的な内省セクション。以下の内容を記述する。
- **分析の限界**: データの期間や品質、モデルの仮定に伴う制約。
- **潜在的リスク**: 検証に含まれていない市場要因（流動性、マクロ変動など）。
- **今後の改善方向**: 次のフェーズで取り組むべき改良点。

### 2.5 視覚化 (Visuals)
- **Mermaid シーケンス図**: トレード執行の論理フローを可視化する。
- **パフォーマンス・プロット**: `plot_alpha.py` により生成された6項目のグラフを引用する。

## 3. テンプレート (Template)

`/templates/report_template.md` をベースとして、各実証実験の成果を反映させること。
