# 自律型クオンツ・ロジック・シーケンス

```mermaid
sequenceDiagram
    autonumber
    actor 人間 as 人間
    participant 統括 as オーケストレータ
    participant 記憶 as 長老
    participant データ as 市場データ基盤エンジニア
    participant 分析 as クオンツ研究者
    participant 執行 as 執行エージェント

    Note left of 統括: フェーズ1: 入力と探索
    人間->>統括: 要件を入力
    統括->>記憶: 履歴を取得
    記憶-->>統括: シード/禁止領域を返却
    統括->>統括: アイデア生成
    統括->>記憶: アイデア候補を保存
    統括->>データ: PIT整合/欠損補完済みデータ作成を依頼
    データ-->>統括: 学習用データセットと文脈を返却
    統括->>統括: データ納入条件を判定
    alt 納入条件未達
        統括->>データ: 再作成を依頼（6へ）
        データ-->>統括: 改訂データセットを返却
        統括->>統括: データ納入条件を再判定（7へ）
    else 納入条件達成
        統括->>統括: 次段へ進行
    end
    統括->>記憶: データ版と前処理条件を保存
    
    Note left of 統括: フェーズ2: 評価と判定
    統括->>分析: 候補式とデータセットを入力
    分析->>分析: 基盤モデルを選定（時系列/テキスト/マルチモーダル）
    分析->>分析: 適応方針を設計（LoRA/プロンプト/制約）
    分析->>分析: 因子探索と候補式の精査を実行
    分析->>分析: 共最適化/バックテストを実行
    分析-->>統括: 採否/主要指標(Sharpe/IC/MDD)/モデル構成を返却
    統括->>統括: 納入条件を判定
    alt 納入条件未達（データ起因）
        統括->>データ: 再作成を依頼（6へ）
        データ-->>統括: 改訂データセットを返却
        統括->>分析: 再検証を依頼（14へ）
    else 納入条件未達（モデル起因）
        統括->>分析: 再選定を依頼（10へ）
        分析-->>統括: 改訂モデル構成を返却
        統括->>分析: 再検証を依頼（14へ）
    else 納入条件達成
        統括->>統括: 次段へ進行
    end
    統括->>記憶: 検証結果とモデル構成を保存
    
    Note right of 統括: 最終判定
    
    alt 採用
        統括->>統括: 発注可否を最終確認（制約/期限/容量）
        alt 発注可
            統括->>執行: 注文を生成
            執行->>執行: 注文を執行し約定を取得
            執行-->>統括: 執行結果を返却
            統括->>記憶: 採用理由と執行結果を保存
        else 発注不可
            統括->>記憶: 発注不可理由を保存
        end
    else 棄却
        統括->>記憶: 棄却理由と主要指標を保存
    end
```

## アイデア
1. 要件と履歴を先にそろえて、探索の無駄を減らす。
2. アイデア候補は早い段階で保存し、再利用可能にする。
3. データ作成条件を保存し、検証の再現性を確保する。
4. クオンツ研究者はモデル選定から検証までを一貫実行する。
5. 採否だけでなく理由と指標を保存し、次回探索に反映する。
