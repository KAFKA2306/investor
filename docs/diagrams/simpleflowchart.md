# 自律型クオンツアルファ生成パイプライン

```mermaid
flowchart TD
    subgraph メタレイヤー [メタレイヤー]
        記憶管理["記憶管理"]
        状態監視["状態監視"]
        要件入力["要件入力"]
        シード選択["シード選択"]
        状態監視 --> |更新| 記憶管理
        記憶管理 --> シード選択
        要件入力 --> シード選択
    end

    subgraph データ理解層 [データ理解]
        データ収集["データ収集（財務/ニュース/SNS/チャート/On-chain）"] --> データ統合["データ統合"]
        データ統合 --> データ受入判定{"データ受入判定"}
        データ受入判定 -- "可" --> シナリオ生成["シナリオ生成"]
        データ受入判定 -- "否" --> データ統合
    end

    subgraph 研究検証フェーズ [研究・検証]
        知識参照["知識参照"] --> 因子探索["因子探索"]
        シード選択 --> |入力| 因子探索
        シナリオ生成 --> |文脈入力| 因子探索
        因子探索 --> |変異/構造更新| 候補生成["候補生成"]
        データ統合 --> 候補生成
        候補生成 -.-> |候補保存| 記憶管理
        候補生成 --> 基盤モデル選定["基盤モデル選定（時系列/テキスト/マルチモーダル）"]
        基盤モデル選定 --> 適応方針設計["適応方針設計（LoRA/プロンプト/制約）"]
        適応方針設計 --> 検証最適化["検証最適化（共最適化/評価/バックテスト）"]
        基盤モデル選定 -.-> |モデル構成保存| 記憶管理
        検証最適化 -- "否" --> 棄却["棄却・記録"]
        検証最適化 -- "可" --> 検証受入判定{"検証受入判定"}
        検証受入判定 -- "可" --> 採用判定{"採用判定（Sharpe/IC/MDD）"}
        検証受入判定 -- "否（データ）" --> データ統合
        検証受入判定 -- "否（モデル）" --> 基盤モデル選定
        検証最適化 -.-> |検証結果保存| 記憶管理
        採用判定 -- "否" --> 棄却
        採用判定 -.-> |採否保存| 記憶管理
        
        棄却 -.-> |学習反映| 記憶管理
    end

    subgraph 執行監査フェーズ [執行・監査]
        採用判定 -- "可" --> 発注ゲート{"発注ゲート"}
        発注ゲート -- "可" --> 配分最適化["配分最適化"]
        発注ゲート -- "否" --> 棄却
        配分最適化 --> リスク制御["リスク制御"]
        リスク制御 --> ヘッジ最適化["ヘッジ最適化"]
        ヘッジ最適化 --> 注文生成["注文生成"]
        注文生成 --> 執行["執行"]
        注文生成 -.-> |注文計画保存| 記憶管理
        執行 -.-> |執行結果保存| 記憶管理
        執行 --> 監査記録["監査記録"]
        監査記録 --> ドリフト分析["ドリフト分析"]
        ドリフト分析 -.-> |監視入力| 状態監視
        監査記録 -.-> |学習入力| 記憶管理
    end

    データ理解層 --> 研究検証フェーズ
    研究検証フェーズ --> 執行監査フェーズ
```

## アイデア
1. 記憶管理を起点にして、探索・検証・執行を循環させる。
2. 発見フェーズで候補を保存し、探索の連続性を確保する。
3. 検証前に基盤モデル選定と適応方針設計を明示する。
4. 検証結果と採否を保存し、判断基準を蓄積する。
5. 注文計画と執行結果を保存し、運用改善に接続する。
